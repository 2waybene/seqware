Metadata - Data Resources

Overview

This sub-domain is exclusively for low-level access of database tables. Each URI actually retrieves one or more rows from a database table. The output format is exclusively XML produced by JAXB.

Retrieving a list of all items in a table is plural e.g. /metadata/db/studies

  • GET: Grouped resources are of the type TableNameList, where TableName is the singular name of the table.
  • POST: Add a new row to the table
  • PUT and DELETE are not permitted

Queries on the table as a whole can be added to the URI as query parameters, e.g. /metadata/db/studies?title=PCSI to find all of the studies with the title of PCSI

Retrieving a particular table row is singular e.g. /metadata/db/study/63

  • GET: returns the table row with the given SWID
  • PUT: updates the row in the table with the row in the request
  • POST and DELETE are not permitted

Additional fields can be shown by adding ?show to the query parameters, e.g. /metadata/db/lanes/1234?show=workflowRun shows the workflow run associated with this lane.

Resources

The Web service that this API refers to is http://localhost:8888/seqware-webservice. For more information about each resource and the correct, complete URI, please follow the link to the resource-specific page.

URI Description
/experiments/(experimentSWID) Retrieves a list of experiments.
/experiments Retrieves one or more experiments, depending on parameters.
/experiments Adds a new experiment to the experiment table.
/files/(fileSWID) Retrieves a file using its SWID.
/files/(fileSWID) Updates a file with the given SWID.
/files Retrieves a list of files.
/experiments Adds metadata about a file to the database.
/ius/(iusSWID) Retrieves an experiment based on the SWID.
/ius/(iusSWID) Update an ius with the given SWID.
/ius Retrieves a list of IUSes.
/lane/(laneSWID) Retrieves a lane with a particular SWID.
/lane/(laneSWID) Updates a lane with the given SWID.
/lanes Retrieves a list of lanes.
/processes/(processSWID) Retrieves a processing object by its SWID.
/processes/(processSWID) Updates a processing object with the given SWID.
/processes Retrieves a list of processings.
/processes Add a new processing object to the database.
/samples/(sampleSWID) Retrieves a sample using its SWID.
/samples Retrieves a list of samples
/samples Add a new sample to the database.
/sequencerrun/(sequencerRunSWID) Retrieves a sequencer run by SWID.
/sequencerruns Retrieves a list of sequencer runs.
/studies Retrieves a list of studies
/studies Adds a new study to the database
/studies/(studySWID) Retrieves a study using its SWID.
/workflows/(workflowSWID) Retrieves a workflow using its SWID.
/workflowparams Retrieves a list of workflow parameters.
/workflowparams Add a new workflow parameter to the database.
/workflowparamvalues Retrieves a list of workflow parameter values.
/workflowparamvalues Adds a workflow parameter value to the database
/workflowruns/(workflowRunSWID) Retrieves a workflow run by SWID
/workflowruns/(workflowRunSWID) Updates a workflow run with the given SWID.
/workflowruns Retrieves a list of workflow runs.
/workflowruns Inserts a new workflow run into the database.
/workflows Retrieves a list of workflows.
/workflows Adds a new workflow to the database.
